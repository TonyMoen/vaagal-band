Status: Ready for Review

## Story

## Implementation Notes for Developer Agent

### Critical Implementation Details

1. **Keep Web3Forms Integration**: The existing onSubmit handler must be preserved. Only the form structure and styling changes.

2. **Subject Field Data**: Add the subject field to FormData:
   ```typescript
   formData.append("subject", selectedSubject)
   ```

3. **Controlled vs Uncontrolled**: The existing form uses uncontrolled inputs with FormData. shadcn Select requires controlled state - add `useState` for subject:
   ```typescript
   const [subject, setSubject] = useState("")
   ```

4. **Form Container**: Keep the existing `card-surface` class on the section element for consistent styling with rest of site.

5. **Hidden Access Key**: The access_key is appended programmatically, not in a visible field - this pattern must remain.

### Potential Gotchas

- shadcn Select returns value differently than native select - use onValueChange prop
- Some shadcn components use `ring-offset-background` which may not exist - override with ring-offset-0
- Button component already exists and works - use it instead of custom `.btn` class
- The existing sweetalert2 (Swal) integration can remain unless user requests change

A professional contact form that:
- Looks consistent with the dark-themed site
- Uses shadcn/ui components throughout
- Has a new Subject dropdown field
- Maintains all existing functionality
- Is fully accessible and keyboard-navigable
